# Milestone v1.1: Structured Q/A Panel

**Status:** ✅ SHIPPED 2026-07-07
**Phases:** 8-9
**Total Plans:** 5 (all complete)

## Overview

A brownfield milestone (after v1.0 shipped all 7 phases) that turns the Q/A panel from one
flat rolling-text blob into a structured, speaker-attributed, Question/Statement-classified
view. Split into a data/seam layer (Phase 8) then the UI redesign that consumes it (Phase 9).

The `ISttProvider` seam was extended to carry speaker + Question/Statement classification
(never coupling consumers to Deepgram); diarization was enabled on the single mixed loopback
stream (no mic); a stable session-scoped `Person N` speaker map absorbs Deepgram's drifting
raw indices; and the left "Q/A" panel was rebuilt in place as a scrollable stack of
per-utterance cards with a compact people-list color legend.

## Phases

### Phase 8: Diarized Utterance Pipeline

**Goal**: The transcript stream stops being one flat text blob and becomes a stream of discrete, speaker-attributed, classified utterances — each carrying a stable `Person N` label and a Question/Statement tag — delivered through the existing `ISttProvider` seam so nothing downstream is coupled to Deepgram. This is the data/seam layer the Phase 9 card UI consumes.
**Depends on**: Phase 7 (v1.0)
**Requirements**: QA-01, QA-02, QA-03, QA-07
**Plans**: 3 plans

Plans:

- [x] 08-01 — Pure core + seam extension: `IUtteranceEvent`/`on('utterance')` on the seam (no Deepgram import), `classifyUtterance` heuristic (D-07/D-08), `pickModalSpeakerIndex` + empty-commit-safe `UtteranceAccumulator`, first-seen `SpeakerMap` with neutral bucket + reset — with all three Wave 0 test files (QA-02, QA-03, QA-07)
- [x] 08-02 — Gateway wiring: `diarize:'true'` + `utterance_end_ms:'1000'` on the live connect, message-type switch, accumulate-then-commit (one `utterance` per `speech_final`, `UtteranceEnd` fallback, no double-commit), `Person N` + Q/S labeling emitted through the seam, D-11 timer audit + regression (QA-01, QA-07)
- [x] 08-03 — Main wiring: additively extend `IOverlayTranscript` with the committed-utterance list on the existing read-only `jedi:transcript` channel (no new control channel), re-key-safe `on('utterance')` binding, and Ctrl+Alt+K reset of buffer + utterance list + Person N numbering together (QA-01, QA-02, QA-07)

**Details:** The `ISttProvider` seam was preserved (QA-07/TRN-05): a sibling `utterance` event carries `speaker` + classification rather than coupling consumers to Deepgram. Diarization enabled via `diarize: 'true'` + `utterance_end_ms: '1000'` on the Deepgram v5 `listen.v1.connect` (no live `utterances` option; `speaker` is per-word at `words[].speaker`, utterance boundaries from `speech_final`/`UtteranceEnd`). The stable speaker map (QA-02) is a main-process session-scoped structure mapping drifting Deepgram indices → stable `Person N`, held alongside `TranscriptBuffer`. Q-vs-statement (QA-03) is a LOCAL heuristic (punctuation `?`, leading interrogatives) — no AI call — defaulting to Statement when unsure.

### Phase 9: Card-Based Q/A Panel Redesign

**Goal**: The Q/A panel (`src/renderer/src/components/transcript-panel.tsx`, the left column) is rebuilt IN PLACE from a single flat text blob into a scrollable stack of per-utterance cards — each labeled `Q1 - Person 1` / `S3 - Person 2`, questions visually distinct from statements — with a compact people list of the identified speakers. The surrounding 4-panel overlay layout is unchanged; only this panel's internals change.
**Depends on**: Phase 8
**Requirements**: QA-04, QA-05, QA-06
**Plans**: 2 plans

Plans:

- [x] 09-01 — Rebuild the panel body: pure `utterance-view.utility.ts` derivation (per-type seq + per-speaker color) + committed cards labeled `Q1 - Person 1` / `S3 - Person 2`, questions visually distinct; `reconcileFinalLog` removed (QA-04, QA-05)
- [x] 09-02 — People row (counted colored chips, `Speaker` excluded) + interim ghost card (replaced-not-accumulated) + empty-state placeholder (QA-06, QA-05)

**Details:** Redesign done IN PLACE — the 4-panel row and the panel's data-testid seams (`card-transcript-panel`, active-indicator, scroll routing via `onScrollTranscript` gated on `activePanel === 'transcript'`) are all preserved. The panel is a pure one-way view (IN-01) consuming the Phase 8 structured utterance stream over the existing read-only bridge; no renderer→main control channel added. Per-type session-scoped sequence numbering (`Q1`, `S3`); per-speaker color via `data-speaker-color` CSS slots (no inline style). A code-review Critical (CR-01: the clear-reset gated on the 90s-rolling `finalText` and wiped the card stack during a lull) was caught and fixed (commit `a60da38`) to gate on `next.utterances.length`.
