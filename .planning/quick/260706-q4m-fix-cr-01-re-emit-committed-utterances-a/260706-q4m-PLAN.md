<objective>
Fix CR-01: the Phase 8 rewrite of `DeepgramSttGateway.handleMessage` re-emits finalized speech ONLY as an `utterance` event and never as a final `transcript` event. Because `index.ts`'s `if (event.isFinal) buffer.appendFinal(event.text)` branch is now dead code, `TranscriptBuffer.finals` stays permanently empty, so `AiOrchestrator.trigger()` always hits the empty-span guard and never calls Claude — a regression of the project's core value (AI-01/AI-02) and of the rolling finalized-transcript display (TRN-01/TRN-02).

The fix (verifier + code-review confirmed): emit an ADDITIVE `{ text: committed.text, isFinal: true }` transcript event from `commitPendingUtterance`, alongside the unchanged `utterance` emit, guarded so an empty/whitespace committed turn emits nothing (WR-01). Because `TranscriptBuffer.appendFinal` clears the interim as a side effect, restoring this emit also resolves the stale-trailing-interim-on-commit warning (WR-02) with no change to `index.ts`.

Purpose: restore the live gateway→buffer→AI-orchestrator span feed that the app's core value depends on, without breaking the D-01 "one utterance per turn" contract or coupling any consumer to `@deepgram/sdk`.
Output: a one-line-of-behavior fix in `deepgram-stt.gateway.ts` plus corrected/added tests that assert the RESTORED behavior (not the regression).
</objective>
