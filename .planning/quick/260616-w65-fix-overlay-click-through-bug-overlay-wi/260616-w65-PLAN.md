<objective>
Fix the overlay click-through bug surfaced during the Phase 1 GO/NO-GO gate (01-04).

The overlay window in `src/main/overlay-window.manager.ts` is created `focusable: false`,
which prevents it from taking *keyboard* focus — but a transparent, always-on-top window
still *captures mouse events* across its entire 420×220 surface. Clicks landing on the
overlay never reach the windows underneath, so the user cannot click anything beneath it.
This violates OVL-02 (the overlay MUST NOT take keyboard/mouse focus from the active app)
and the project's CLAUDE.md "load-bearing" requirement of `setIgnoreMouseEvents(true, { forward: true })`.

The app is keyboard-only, so click-through must be permanent.

This is a gap-closure fix on plan 01-02's deliverable. It does NOT close plan 01-04 — the
user must re-run the on-machine GO/NO-GO gate after this fix.
</objective>
