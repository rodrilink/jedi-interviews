# Quick Task 260616-w65: Fix Overlay Click-Through Bug Summary

Made the always-on-top overlay click-through with `setIgnoreMouseEvents(true, { forward: true })` so mouse clicks pass to the windows beneath it, closing the OVL-02 gap that `focusable: false` (keyboard-focus only) left open.

## What Changed

Single task, single file (`src/main/overlay-window.manager.ts`):

1. **`createOverlayWindow()`** — Added `window.setIgnoreMouseEvents(true, { forward: true })` immediately after the `new BrowserWindow({...})` call, with a comment explaining that `focusable: false` only governs keyboard focus while this call is what stops the transparent overlay from capturing mouse clicks across its whole surface. `forward: true` preserves move events for the renderer.

2. **`showOverlay()`** — Re-asserted the same `setIgnoreMouseEvents(true, { forward: true })` call alongside the existing `setAlwaysOnTop` / `setContentProtection` re-applications, for the same defensive reason: the OS can reset window attributes across hide/show cycles, so click-through is re-applied on every show.

3. **TSDoc** — Updated `createOverlayWindow`'s doc comment to document the click-through behavior and WHY (clicks must reach the window beneath; click-through is permanent because the app is keyboard-only), and corrected the previous "keyboard/mouse focus" wording to clarify that `focusable: false` governs keyboard focus only — mouse click-through is the separate concern this fix addresses. Also extended `showOverlay`'s TSDoc to note click-through is re-applied on every show.

## Why

A transparent, always-on-top window still captures mouse events across its entire 420×220 surface even when `focusable: false` prevents it from taking keyboard focus. Clicks landing on the overlay never reached the windows underneath, so the user could not click anything beneath it — violating OVL-02 (the overlay MUST NOT take keyboard/mouse focus from the active app) and the project's CLAUDE.md "load-bearing" requirement of `setIgnoreMouseEvents(true, { forward: true })`. The app is keyboard-only, so click-through is permanent.
