| settings renderer → main (saveContext DTO) | Untrusted DTO shape is persisted; validation happens at the IPC handler (06-04), repository assumes a typed DTO |
| main → disk (electron-store) | Non-secret context JSON persisted under userData |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Tampering | saveActive DTO shape | mitigate | Repository takes a typed `fields` object (not raw renderer input); the IPC handler in 06-04 validates DTO shape before calling saveActive |
| T-06-08 | Information Disclosure | context contents (notes/ticket) | accept | Session context is non-secret user-pasted text; stored plaintext under userData by design (CTX-02). No PII/credential handling here |
| T-06-09 | Denial of Service | unbounded context growth | accept | v1 single-context UI persists one DTO; no unbounded array. Multi-context is schema-only (D-09) |
