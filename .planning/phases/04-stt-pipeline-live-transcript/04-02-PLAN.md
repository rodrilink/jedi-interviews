## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process -> Deepgram (network egress) | 16 kHz mono PCM audio leaves the machine to api.deepgram.com |
| Deepgram API key -> process memory | Secret read from env/dev-config, used to authenticate the socket |
| Deepgram message payload -> gateway | Untrusted remote JSON parsed into transcript events |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-02 | Information Disclosure | Deepgram API key | mitigate | Key read in main only (env/dev-config, D-08); never `console.log`'d, never in an emitted payload, never crosses IPC; source-assert no log call takes the key; do not log full request URLs that could embed it |
| T-4-03 | Information Disclosure | Audio sent to Deepgram (privacy) | accept | Accepted for v1 per PROJECT constraints; documented; Whisper-local is the v2 privacy fallback behind the same ISttProvider seam |
| T-4-04 | Tampering | Untrusted Deepgram `message` payload controlling flow | mitigate | Defensive optional-chaining (`message.channel?.alternatives?.[0]?.transcript ?? ''`) + empty-text guard; payload only produces render-text, never control flow |
| T-4-05 | Denial of Service | Socket idle-close during silence -> reconnect churn | mitigate | `sendKeepAlive({})` timer during silence (Pitfall 3); bounded exponential backoff with jitter prevents reconnect storms; gateway never throws on transport error |
</threat_model>
