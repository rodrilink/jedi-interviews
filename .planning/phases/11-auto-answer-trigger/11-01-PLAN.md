<objective>
Wire the auto-answer trigger end-to-end on the main side: when the live utterance stream classifies a committed turn as a question, automatically enqueue a grounded `answer` request into the Phase 10 priority queue's `'auto'` lane — no keypress — so the same answer content the manual `Ctrl+Alt+A` path produces streams into the existing AI panel. Reconcile Phase 10's mode-keyed burst debounce (D-06) so distinct questions each answer while an identical repeated question within the window collapses (D-01). Reorder boot construction so the orchestrator exists before `wireSttPipeline` (D-03). Add a `source` field to the `thinking` push so a downstream renderer plan can badge auto entries (D-04, main side only here).

Purpose: This is the vertical slice that closes the v1.2 auto-answer loop (AA-01, AA-02). It deliberately reverses the v1 "AI calls are user-triggered only" constraint; cost stays bounded by the Phase 10 debounce + single-in-flight + bounded cap (the Phase 12 Off mode lands later).
Output: A working main-side auto-trigger with content-dedup, grounding parity, and a `source`-carrying thinking push, all covered by unit tests.
</objective>
