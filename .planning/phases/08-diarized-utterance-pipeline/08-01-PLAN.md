<objective>
Build the three pure/testable core units of the diarized-utterance pipeline and extend the STT seam contract to carry a committed utterance — before any Deepgram socket wiring depends on them (interface-first ordering).

This is the innermost vertical slice: given a run of diarized words and text, the pipeline can (a) group them into one utterance and pick the modal speaker index, (b) map that index to a stable `Person N` (or a neutral bucket when undiarized), and (c) classify the text Question/Statement — all as pure, socket-free, unit-tested logic behind the extended `ISttProvider` seam. Wave 2 (Plan 02) wires these into the live gateway.

Purpose: Satisfies QA-02 (stable speaker map), QA-03 (local Q/S heuristic), and the contract half of QA-07 (seam extension), keeping all logic pure and injectable per D-10 so it is unit-tested without a live socket. Closes the Wave 0 test gaps for these three files (Nyquist Dimension 8).
Output: `IUtteranceEvent`/`UtteranceClassification` on the seam, three new source files, and three co-located `*.test.ts` files, all green.
</objective>
