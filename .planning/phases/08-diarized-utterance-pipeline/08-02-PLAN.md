<objective>
Wire the Plan 01 units into the live Deepgram v5 socket so the gateway emits discrete, speaker-attributed, classified `utterance` events through the `ISttProvider` seam — while keeping the D-02 interim `transcript` line intact and hardening every timer callback against a closed-socket throw (D-11).

This vertical slice moves a Deepgram payload end-to-end through the seam: raw `message` → defensive parse → accumulate is_final runs → commit one utterance on `speech_final` (or `UtteranceEnd` fallback) → modal speaker index → `Person N` label + Q/S classification → typed `utterance` seam event. All `@deepgram/sdk` coupling stays inside this one file (D-09/QA-07).
