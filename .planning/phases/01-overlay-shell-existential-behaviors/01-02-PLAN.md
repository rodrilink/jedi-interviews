<objective>
Build the overlay WindowManager: a transparent, frameless, always-on-top, focusable:false BrowserWindow shown only via a `showOverlay()` wrapper that re-applies content protection on every show (and re-asserts on blur/display change), rendering a toggleable debug HUD that displays Electron version, content-protection state, and window position via a read-only non-secret preload status channel. This is the visible end-to-end slice of the Walking Skeleton: real main → preload → renderer data flow.

Purpose: These are the two existential behaviors (never steal focus, invisible to capture) plus the self-evidencing readout used during the GO/NO-GO gate (01-04). The HUD doubles as the on-screen verification surface.
Output: An overlay window + HUD that, when shown, floats over everything, never grabs focus, and (with the right Electron patch) is absent from screen capture. Covers OVL-01, OVL-02, OVL-04.
</objective>
