<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main process ↔ OS keystore (DPAPI) | safeStorage encrypt/decrypt crosses to the OS-backed key store; main-only. |
| persisted store (userData) | Ciphertext at rest on disk; must never contain plaintext. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-I | Information Disclosure | renderer reads secrets | mitigate | No secret-bearing IPC/contextBridge in Phase 1 (D-05); round-trip is main-only. Renderer cannot reach safeStorage. |
| T-01-03-I2 | Information Disclosure | plaintext persisted/committed | mitigate | DPAPI `encryptString`; persist only base64 ciphertext (D-04); placeholder is fake; store file under userData (outside repo) — gitignore any local copy. |
| T-01-03-I3 | Information Disclosure | secret written to logs | mitigate | Never log plaintext or decrypted value; verify:secret prints only PASS/FAIL (V7). |
| T-01-03-T | Tampering | stored ciphertext corrupted | mitigate | DPAPI `decryptString` fails closed; the equality assertion in proveSecretBoundary detects corruption → verify:secret returns FAIL. |
| T-01-03-E | Elevation of Privilege | renderer → Node to reach safeStorage | mitigate | Inherits contextIsolation+sandbox boundary from 01-01; safeStorage never imported into preload/renderer. |
</threat_model>
